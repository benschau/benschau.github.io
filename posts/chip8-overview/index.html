The Quickest, Worst Chip8 Overview

Share

Who is this for?

For people that want a quick overview of the basic stuff you need to know how to write a CHIP-8 interpreter.

What is CHIP-8?

CHIP-8 is an interpreted programming language & virtual machine made in the 70s. More importantly, it’s a fun little thing to do if you’re bored and want to do something related to the low-level side of computing.

The CHIP-8 Hardware Stuff

The CHIP-8 is an 8-bit system that consists of a 4Kb memory, 16 registers, a stack, two timers, a hex keyboard (16 inputs) and a display of 64x32. More interestingly, it also supports sprites (think 2D bitmaps you can just draw on top of things in bulk) and has its own font set. The PC (program counter) register and I (address register) are separate from the register file.

CHIP-8 Instruction Set

The CHIP-8 instruction set is made up of 35 ops, of which you can find listed on Wikipedia. It’s a handy reference when you’re writing the instruction set opcode by opcode.

Each opcode is made up on 2 bytes for a total of 16 bits in big-endian. This means that the higher order bits come first in the byte order- e.g if I read in 0x12 and 0x34 sequentially, the big-endian opcode is interpreted as 0x1234. In little endian, the lower order bits come first, so you should interpret it as 0x3412 instead. (NOTE: Endianness in bit order is a thing also.)
