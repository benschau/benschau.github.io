A Vim Primer

This one’s for my IDE friends that can’t exit Vim.

What is Vim?

Vim (Vi-iMproved) is a free and open-source text editor. That’s it, really!

Why Vim?

I’m glad you asked, here’s the short version:

  1. Vim is extendable: Vim has a billion add-ons, everything from code completion to LateX previewers are available. You can contribute yourself if you want to write custom plugins, but the point here is that Vim has a massive amount of support, which will probably continue because …

  2. Vim is everywhere: For those of us that work on low-level systems, there’s no way you don’t use some UNIX based system (and for those of us that don’t, you should; more on this another day). Vim is basically on all these systems (if not Vim, then the original Vi). Learning Vim is especially good if you’re in a pinch, since it’s pretty much guaranteed to be around in some form. Moreover, plugins for Vim exist for virtually every popular text editor (VS code & Atom at the very least). Hell, I use some Vim keybindings on Firefox.

  3. Vim is lightweight: Vim is everywhere since Vim is very, very small. v8.1.1239 is at least 48.8 MB, as compared to Emacs v26.2 at 224.6 MB- a little less than 5 times as large. Of course, there’s a good reason for all that (Emacs as a result can act much more like an IDE than Vim), so it depends on what you want. That being said, in my experience Vim allows fast navigation and key-stroke efficiency with what it does have.

And of course, with the rest of this intro, I would recommend getting your hands on Vim in some way. I’ll leave that part to you- just Google it.

The Basics

So you’ve got your hands on Vim. Let’s start with the basic keystrokes to get you on your way.

Vim has three different modes:

  • normal: The one you’re in when you start the program (ala vim).

  • insert: Hit ‘i’ or ‘a’ while in normal mode to enter insert mode.

    • You can’t move your cursor willfully across lines in this mode- exit to normal to be able to use your arrows/hjkl again.
    • ‘i’ inserts where your cursor is at, and ‘a’ moves your cursor to the right and enters insert mode.

  • visual: Hit ‘v’ while in normal mode to enter visual mode.

    • Also known as the ‘select’ mode- think of highlighting the text with your mouse, but in Vim this is done via your cursor.

In insert and visual modes, hit ‘esc’ to return to normal mode.

While in normal and visual mode, you can move your cursor around using the arrow keys or hjkl. You can also delete individual characters while in normal mode via ‘x’, which will delete the character your cursor is at. Insert mode is for, well, inserting text (go ahead and hit the keys to get some actual text).

Once you’ve finished writing whatever it is that you want to write to the file, hit ‘ZZ’ to save and quit.

  • ‘:wq’ will also save and quit. You can also use ‘:q!’ to force quit without saving.

  • If you entered Vim via vim, you probably haven’t specified a file to write to- hit ‘:w filename’ to specify one.

    • For future reference, you can specify vim filename to avoid having to do that.

Navigation

Beyond just moving around in that one text file you have, Vim can create tabs and split windows (horizontal & vertical) by default.

If you want to create tabs before you’ve entered Vim, you can use the ‘-p’ argument, ala vim -p fname1 fname2 to create two separate tabs. Otherwise, you can use ‘:tabedit fname1’ to open a file in a tab.

To switch between the tabs, go to the next tab using ‘gt’, the previous one with ‘gT’, and a i (integer) indexed tab like ‘2gt’ (tab at position 2, and for our CS majors, not zero-indexed).

You can close tabs using ‘:tabclose’ and a i indexed tab via ‘:tabclose i’.

To split windows, you can perform a vertical split via ‘:vs fname1’ and a horizontal split via ‘:split fname1’. To switch between windows, hit ‘Ctrl-W’ and a direction (hjkl or arrows) to switch to that window.

Conclusion

That’s all she wrote. These are the basics just to get you started, but you’ll eventually want more (as I promised in the Why Vim? section). Here are a couple things to look at if Vim is what you’re looking for.

  • Configuration: See your .vimrc- if you don’t have one and you’re on a Linux/Unix based system, it’s usually located in the home directory. Here, you can place vimscript stuff that Vim will automatically run when you start the editor. If you want a good example of what’s possible, check out amix’s vimrc.

  • Nouns, Verbs, Modifiers: the Vim grammar, not to be confused with vimscript. For instance, try writing out a paragraph and hitting ‘dap’. This will delete an entire paragraph (or what Vim thinks is a paragraph) and enter insert mode. In this case, ’d’ is our verb, ‘a’ is our modifier, and ‘p’ our noun.

    • One resource I found especially helpful for learning this is Yan Pritzker’s Learn to speak vim - verbs, nouns, and modifiers!. It’s amazing. (Question: what do you think ‘yap’ does?)

Thanks for reading. Hopefully by now, you’re at least able to navigate around Vim as if it were Notepad. Learning Vim is a process, but if you stick with it, you’ll find it’s actually very, very intuitive.
